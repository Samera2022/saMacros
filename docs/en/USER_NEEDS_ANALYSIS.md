# saMacros User Needs Analysis

## 📊 Why Do Users Need This Mouse Macro Tool?

### Core Problem: The Dilemma of Repetitive Labor

In digital work and entertainment, users face numerous **repetitive and mechanical operation tasks** that:
- ⏰ **Waste Time**: May require hours daily performing identical operations
- 😫 **Cause Fatigue**: Repetitive clicking and mouse movements lead to physical exhaustion (RSI - Repetitive Strain Injury)
- 😤 **Reduce Efficiency**: Manual operation speed is limited, affecting overall work efficiency
- 😔 **Prone to Errors**: Prolonged repetitive operations lead to decreased attention and mistakes

## 🚀 Advanced Features Meet Advanced Needs

### 1. **Script System - Beyond Simple Recording**
```javascript
// Example: Respond differently based on screen color
mm.on('io.github.samera2022.samacros.api.event.events.AfterRecordStopEvent', (event) => {
    let color = mm.getContext().getPixelColor(100, 200);
    if (color.getRed() > 200) {
        mm.log('Red detected, execute action A');
        // Execute specific operation
    } else {
        mm.log('Red not detected, execute action B');
        // Execute other operation
    }
});
```

**Application Scenarios:**
- Determine if game has reached specific location based on screen
- Execute different operations based on software state
- Implement intelligent conditional logic

### 2. **Event-Driven - Respond to Application State**
```javascript
// Listen to application launch event
mm.on('io.github.samera2022.samacros.api.event.events.OnAppLaunchedEvent', (event) => {
    mm.log('Application launched, auto-loading configuration');
    // Automatically load last saved macro
});

// Listen to recording complete event
mm.on('io.github.samera2022.samacros.api.event.events.AfterRecordStopEvent', (event) => {
    mm.log('Recording complete, captured ' + event.getActions().size() + ' actions');
    // Auto-save or analyze recorded content
});
```

**Application Scenarios:**
- Automated workflow integration
- Monitoring and logging
- Dynamic response to application state

### 3. **Plugin Ecosystem - Community Power**
- Users can share scripts they've written
- Support dependency management (hard and soft dependencies)
- Whitelist mechanism ensures security
- Authors can build complex automation suites
