# 新增付费功能 5 分钟接入模板

> 适用范围：在现有“统一付费功能网关”架构下，快速给新功能接入付费校验。  
> 目标：只改少量固定位置，不改加密存储主流程。

---

## 0. 前置条件（30 秒）

确认以下基础已存在：

- `LicenseFeature`（能力枚举）
- `LicenseGateService`（统一网关）
- `LicenseIntegrityGuard`（完整性可信判定）
- 目标功能有明确的 **core 真实执行入口**（不是仅 GUI 按钮）

---

## 1. 新增能力点（1 分钟）

文件：

- `samacros-core/src/main/java/io/github/samera2022/samacros/core/license/LicenseFeature.java`

操作：

- 新增一个能力枚举，例如：`ADVANCED_PLAYBACK`

示例：

```java
public enum LicenseFeature {
    SCRIPT_MANAGER,
    SCRIPT_EXECUTION,
    ADVANCED_PLAYBACK,
    RESERVED_FEATURE_1,
    RESERVED_FEATURE_2
}
```

---

## 2. 配置 entitlement 映射（1 分钟）

文件：

- `samacros-core/src/main/java/io/github/samera2022/samacros/core/license/LicenseGateService.java`

操作：

- 在 `ENTITLEMENT_ALLOWED_FEATURES` 和/或 trial/valid 映射中加入新能力。

示例（仅示意）：

```java
private static final Map<String, EnumSet<LicenseFeature>> ENTITLEMENT_ALLOWED_FEATURES = Map.of(
    "PERSONAL", EnumSet.of(LicenseFeature.SCRIPT_MANAGER, LicenseFeature.SCRIPT_EXECUTION),
    "COMMERCIAL", EnumSet.of(LicenseFeature.SCRIPT_MANAGER, LicenseFeature.SCRIPT_EXECUTION, LicenseFeature.ADVANCED_PLAYBACK),
    "ENTERPRISE", EnumSet.of(LicenseFeature.SCRIPT_MANAGER, LicenseFeature.SCRIPT_EXECUTION, LicenseFeature.ADVANCED_PLAYBACK)
);
```

---

## 3. 在 core 真实执行入口接入网关（2 分钟）

文件：

- 你的功能实际执行类（示例：`xxx/core/.../AdvancedPlaybackService.java`）

操作：

1. 调用 `LicenseGateService.checkFeature(LicenseFeature.ADVANCED_PLAYBACK)`
2. 若拒绝则立即返回，不继续执行业务
3. 记录 feature / reason / status 日志

示例：

```java
LicenseCheckResult gate = LicenseGateService.checkFeature(LicenseFeature.ADVANCED_PLAYBACK);
if (!gate.isAllowed()) {
    LogManager.log("[AdvancedPlayback] denied: feature=" + gate.getFeature()
        + ", reason=" + gate.getDenyReason()
        + ", status=" + gate.getLicenseStatus());
    return false; // 或抛业务异常
}

// 通过后继续执行
```

---

## 4. GUI 只做提示（1 分钟）

文件：

- 对应 GUI 入口（按钮/菜单）

操作：

- GUI 可以调用同一个 `checkFeature(...)` 来决定提示文案
- 但 GUI 不作为最终安全边界（core 必须已接入）

示例：

```java
LicenseCheckResult gate = LicenseGateService.checkFeature(LicenseFeature.ADVANCED_PLAYBACK);
if (!gate.isAllowed()) {
    // 弹购买/激活提示
    return;
}
// 进入功能页（实际执行仍由 core 再次校验）
```

---

## 5. 验收（30 秒）

命令：

```bash
mvn -pl samacros-core,samacros-gui -DskipTests compile
```

人工检查：

- 未授权账号：GUI 提示 + core 拒绝执行
- 已授权账号：功能可执行
- 日志包含拒绝原因（`DenyReason`）
- 完整性异常场景：返回 `INTEGRITY_CHECK_FAILED`，禁止放行付费能力

---

## 6. 常见错误（避免踩坑）

- 只在 GUI 判断，没在 core 入口判断
- 新增了 `LicenseFeature`，但没更新 `LicenseGateService` 映射
- 通过旧的状态字符串直接写 `if`，没走统一网关
- 拒绝分支没有日志，后续排障困难

---

## 7. 建议命名规范

- `LicenseFeature` 枚举值统一使用大写下划线，如：`ADVANCED_PLAYBACK`
- 日志前缀统一 `[FeatureName]`，便于检索
- 所有新付费能力都走 `checkFeature(...)`，禁止旁路判断

---

## 8. 下一阶段对接 Rust（预留）

当前模板适配 Java 侧网关；未来迁移到 Rust 后：

- 业务入口调用保持不变（仍用 `checkFeature(...)`）
- 仅替换 `LicenseGateService` 的底层判定来源（能力快照改由 Rust 提供）

这能保证“上层接入代码不重写，底层安全能力逐步升级”。
