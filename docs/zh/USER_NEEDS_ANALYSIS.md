# saMacros 用户需求分析

## 📊 为什么用户需要这款鼠标宏工具？

### 核心问题：重复性劳动的困境

在数字化工作和娱乐中，用户面临着大量**重复性、机械性的操作任务**，这些任务：
- ⏰ **耗费时间**：每天可能需要花费数小时进行相同操作
- 😫 **令人疲惫**：重复点击、移动鼠标导致身体疲劳（RSI - 重复性劳损）
- 😤 **降低效率**：手动操作速度有限，影响整体工作效率
- 😔 **容易出错**：长时间重复操作导致注意力下降，出现失误

---

## 🎯 目标用户群体

### 1. **办公自动化用户** 📋
**典型场景：**
- 数据录入员需要在多个系统间复制粘贴数据
- 财务人员需要批量处理发票、报表
- 客服人员需要重复回复相似问题
- HR 需要批量筛选简历、填写表格

**痛点：**
- 每天重复相同的点击、输入、复制粘贴操作数百次
- 不同系统间切换繁琐
- 手动操作速度慢且易出错

**saMacros 解决方案：**
- ✅ 录制一次操作流程，无限次回放
- ✅ 支持跨应用程序自动化
- ✅ 精确到像素的操作重现
- ✅ 可设置循环次数，批量处理任务

### 2. **游戏玩家** 🎮
**典型场景：**
- RPG 游戏需要重复刷怪、采集资源

- 网页游戏需要定时签到、领取奖励
- 竞技游戏需要练习固定技能连招

**痛点：**
- 重复性操作占据大量游戏时间
- 某些操作需要精确时间间隔（如技能连招）
- 长时间操作导致手腕疲劳
- 需要挂机但不想浪费电费和设备寿命

**saMacros 解决方案：**
- ✅ 精确的时间控制（毫秒级延迟）
- ✅ 支持鼠标移动、点击、滚轮、键盘组合操作
- ✅ 暂停/恢复功能，随时干预

**saMacros 解决方案：**

## 🆚 与其他解决方案的对比

### 为什么不用内置的宏功能？
❌ **内置宏功能的局限：**
- 只能在单一软件内使用
- 功能简单，只支持基础操作
- 无法跨应用程序自动化
- 不支持复杂逻辑和条件判断

✅ **saMacros 的优势：**
- 全局热键，任何应用都可使用
- 支持鼠标、键盘、滚轮全面操作
- 脚本系统支持复杂逻辑
- 跨应用程序工作流自动化

### 为什么不用商业 RPA 工具？
❌ **商业 RPA 工具的问题：**
- 价格昂贵（年费数千到数万元）
- 学习曲线陡峭
- 过于复杂，简单任务也需要繁琐配置
- 需要企业级部署

✅ **saMacros 的优势：**
- 简单易用，录制即可使用
- 轻量级，单个可执行文件
- 个人用户和小团队的理想选择

### 为什么不用其他免费宏工具？
❌ **其他免费宏工具的不足：**
- 功能单一，只能录制回放
- 不支持脚本编程
- 界面老旧，不支持高 DPI
- 缺乏维护，兼容性差
- 安全性不明，可能包含恶意代码

✅ **saMacros 的优势：**
- 现代化 UI，支持暗色模式和高 DPI
- 强大的脚本系统（GraalVM JavaScript）
- 积极维护，持续更新
- 多语言支持（7+ 种语言）

## 🚀 进阶功能满足高级需求

### 1. **脚本系统 - 超越简单录制** 
```javascript
// 示例：根据屏幕颜色做出不同响应
mm.on('io.github.samera2022.samacros.api.event.events.AfterRecordStopEvent', (event) => {
    let color = mm.getContext().getPixelColor(100, 200);
    if (color.getRed() > 200) {
        mm.log('检测到红色，执行动作 A');
        // 执行特定操作
    } else {
        mm.log('未检测到红色，执行动作 B');
        // 执行其他操作
    }
});
```

**应用场景：**
- 根据游戏画面判断是否到达特定地点
- 根据软件状态执行不同操作
- 实现智能条件判断

### 2. **事件驱动 - 响应应用状态**
```javascript
// 监听应用启动事件
mm.on('io.github.samera2022.samacros.api.event.events.OnAppLaunchedEvent', (event) => {
    mm.log('应用启动，自动加载配置');
    // 自动加载上次保存的宏
});

// 监听录制完成事件
mm.on('io.github.samera2022.samacros.api.event.events.AfterRecordStopEvent', (event) => {
    mm.log('录制完成，共捕获 ' + event.getActions().size() + ' 个动作');
    // 自动保存或分析录制内容
});
```

**应用场景：**
- 自动化工作流集成
- 监控和日志记录
- 动态响应应用状态

### 3. **插件生态 - 社区力量**
- 用户可以共享自己编写的脚本
- 支持依赖管理（硬依赖和软依赖）
- 白名单机制保证安全性
- 作者可以构建复杂的自动化套件
